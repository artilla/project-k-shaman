## PostgreSQL 16 migration contract

macOS host의 PostgreSQL 16은 ARM64 host에서 x86_64 Homebrew binary로 설치돼 있다.
수동 `initdb`는 System V shared-memory allocation failure로 cluster를 만들지 못했다.
이는 SQL 또는 runner 실패가 아니지만 host aggregate gate가 DB suite를 skip하는
원인이므로, skip 결과를 migration 증거로 인정하지 않았다.

대신 공식 `postgres:16-bookworm` ARM64 container에 repository를 read-only mount하고
다음 계약을 강제 실행했다.

```text
REQUIRE_PG16=1 bats --formatter tap tests/db-migrate.bats
1..70
70 ok
0 not ok
0 skip
```

Container는 테스트 종료 후 제거됐고 repository migration과 runner blob은 변경되지
않았다.
