01한눈에 보기

오늘신당은 프런트엔드와 백엔드를 별도 서비스로 배포하지 않는다. 하나의 image와 하나의 HTTPS origin을 사용하고, 서버 내부만 명시적 계층으로 나눈다.

Deployment

하나의 배포 단위

React production build와 Python wheel을 하나의 non-root image에 넣고 같은 revision으로 배포한다.

Boundary

내부 계층은 강제

domain, application, adapters, web의 의존성 방향을 AST 테스트가 검사한다.

Evolution

포트 뒤 구현 교체

현재 메모리 어댑터는 PostgreSQL·Redis 구현으로 교체할 수 있지만 유스케이스와 route 계약은 유지한다.

왜 마이크로서비스가 아닌가? 현재는 한 팀, 한 배포, 한 데이터 경계다. 독립 확장·소유권 요구가 생기기 전에는 네트워크 분리보다 모듈 경계와 테스트가 더 낮은 비용으로 같은 목적을 달성한다.

02내부 아키텍처

의존성은 바깥 기술에서 안쪽 정책으로만 향한다. 구현 선택은 bootstrap.py 한 곳에서 수행한다.

web
FastAPI route, JSON 검증, cookie, HTTP status, rate/login gate
입력 어댑터
adapters
OAuth HTTP, TTS, cache, session, event JSONL, audio 저장 구현
기술 구현
application
Playback 유스케이스, cache 원자성, telemetry, Protocol 포트
오케스트레이션
domain
운세·꿈·낭독·공유 카드 규칙. HTTP, 환경, 파일, clock 직접 접근 금지
순수 정책
bootstrap
Settings를 받아 실제 adapter와 PlaybackService를 AppContainer로 조립
composition root

강제되는 규칙

Domain 순수성

os, pathlib, sys, 네트워크 client와 직접 clock 접근을 금지한다. input, open, print도 사용할 수 없다.

구현 역참조 금지

application은 adapters/web/bootstrap을 import하지 않고, web은 adapter 구현을 직접 import하지 않는다.

한 곳에서 조립

구체 adapter class는 bootstrap.py에서만 연결된다. 테스트는 포트를 통해 fake 구현을 주입할 수 있다.

설치 의미 일치

src layout, editable install, production wheel이 같은 shindang import 의미를 갖는다.

03주요 요청 흐름

오늘의 운세

React가 POST /api/fortune/today로 topic·profile 입력을 보낸다.
web이 JSON 크기·타입·실제 생년월일을 검증하고 hourly/daily limit을 적용한다.
PlaybackService가 용도 분리 HMAC seed와 cache key를 만들고 cache miss를 직렬화한다.
domain이 운세, 점수, 8개 narration segment를 결정적으로 생성한다.
텍스트 응답을 먼저 반환한다. 실 TTS는 로그인 후 POST /api/tts/prepare에서만 선택적으로 실행한다.

OAuth session

/api/auth/login/google|kakao가 서명 전 state cookie와 provider authorize URL을 만든다.
callback이 state를 constant-time 비교하고 provider의 안정적인 subject가 없으면 fail-closed 처리한다.
성공 시 in-memory session을 만들고 Secure 환경 cookie를 설정하며 state cookie는 즉시 제거한다.
/api/auth/me와 /api/auth/logout이 브라우저 session 상태를 제공한다.

꿈 풀이와 관측

꿈 풀이

로그인·rate/daily gate 뒤 순수 domain.dream이 text와 symbol을 분석하고 SVG 공유 카드를 생성한다.

재생 telemetry

/api/event가 유한 숫자 timestamp와 event object를 검증한 뒤 JSONL adapter로 기록한다.

05런타임과 보안 경계

경계현재 구현효과
OriginFastAPI가 API와 frontend/dist를 함께 제공CORS와 cross-site cookie 복잡도 제거
EdgeCaddy만 80/443 공개, app 8000은 Compose 내부app 직접 접근 차단
Client IPCaddy가 X-Forwarded-For {client_ip} 한 값으로 정규화클라이언트 XFF spoof가 rate identity를 바꾸지 못함
ContainerUID/GID 10001, read-only rootfs, capability drop침해 시 쓰기·권한 범위 축소
Writable state/app/state volume과 /tmp tmpfsevent·TTS cache만 제한적으로 기록
Secretlocal env 또는 AWS Secrets Managerimage·Compose·deployment record에 값 미포함
Imagedigest-pinned base, hash-locked Python, immutable ECR digest동일 revision 재현성과 mutable tag 차단
현재 DB 상태: migration runner와 SQL 계약은 구현돼 있지만 FastAPI runtime DB adapter는 아직 없다. 로컬 mock 실행에서는 DATABASE_URL을 unset해야 한다. 값만 넣으면 /readyz가 의도적으로 503을 반환한다.

10문제 해결

프런트엔드는 열리지만 API가 실패한다
FastAPI가 정확히 8788에서 실행 중인지 lsof -nP -iTCP:8788 -sTCP:LISTEN과 curl http://127.0.0.1:8788/healthz로 확인한다.
/readyz가 503을 반환한다
응답이 adapter-unavailable이면 현재 shell의 DATABASE_URL을 unset하고 FastAPI를 재시작한다.
FastAPI 포트의 /가 404다
Vite 개발 모드에서는 5173으로 접속한다. FastAPI가 SPA까지 제공해야 하면 npm --prefix frontend run build 후 서버를 재시작한다.
OpenAI key 오류로 기동하지 않는다
일반 QA는 export TTS_BACKEND=mock과 unset OPENAI_API_KEY로 되돌린다. 실 TTS는 승인된 비용 범위에서만 실행한다.
5173·8788·7474 포트가 이미 사용 중이다
lsof -nP -iTCP:<port> -sTCP:LISTEN으로 owner process를 확인한다. Vite/FastAPI는 proxy 계약 때문에 5173/8788 유지가 안전하다.

11현재 제약과 확장 경계

인메모리 session/rate limit

재시작 내구성과 다중 instance 일관성이 없다. 수평 확장 전에 PostgreSQL/Redis adapter로 교체한다.

Runtime DB adapter 미구현

migration 안전성은 구현됐지만 application persistence는 T029 범위다. DB 값을 먼저 연결하지 않는다.

단일 EC2 topology

클로즈드 베타의 의도적 trade-off다. 자동 복구·부하·가용성 요구가 확인되면 managed runtime을 재평가한다.

TTS cache와 event file

현재 local volume/JSONL 경계다. multi-instance나 장기 보존 요구가 생기면 object storage·analytics pipeline으로 교체한다.

확장 원칙: 먼저 application port 뒤 adapter를 교체한다. 조직·배포·부하 증거가 생기기 전에는 domain 기능을 별도 네트워크 서비스로 분리하지 않는다.
