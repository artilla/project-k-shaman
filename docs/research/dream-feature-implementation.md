# 꿈 해몽 기능 구현 조사

> 조사일: 2026-07-09 · 원본: `reference/design-prototype-dream/오늘신당 꿈 해몽.dc.html` (claude.ai 디자인)
> 결정 사항(사용자 확정): 진입점은 디자인 에이전트 제안 채택, 입력은 텍스트+상징 칩, **해몽 자체가 로그인 필요**.

## 1. 프로토타입 구조

- **플로우**: E0(S3 주제 선택, 6번째 카드) → [비로그인 시 게이트 모달] → D1 꿈 입력 → D2 해몽 공연 → D3 꿈 부적. 내비는 스택(push/pop) + 로고=홈 — 기존 v2 내비 규칙과 동일.
- **진입 카드**: "꿈 해몽 NEW · 간밤의 꿈을 풀어드려요 · 로그인" — 금선+보라 그라데 하이라이트로 일반 주제와 구분, 로그인 라벨로 게이트 예고. 하단 안내 "일일 운세와 별도로 하루 여러 번 볼 수 있어요".
- **D1**: textarea 300자(카운터, 300자 도달 시 핑크) + 상징 칩 8종(최대 3개 선택, `.slice(-3)`) + 프라이버시 박스("해몽 생성에만 사용되고 서버에 저장되지 않아요. 해몽이 끝나면 이 기기에서도 바로 지워집니다") + CTA "홍연에게 해몽 청하기" (텍스트 or 칩 하나라도 있으면 활성).
- **D2**: S4 스테이지 변형 — 부적 프레임 배경이 보라 틴트(`rgba(123,44,191,.4)`), 아바타 3상태(idle/speaking/blessing). 해몽 카드 = 헤드라인(따옴표 인용) + 상징별 해석(칩 라벨+의미) + 전체 풀이 + "오늘 운세" 연결 한 줄. 플레이어는 기존 패턴(듣기/정지/다시 듣기/꿈 부적 받기).
- **D3**: 꿈 부적 카드 — 보라 계열 그라데(`#1E1233→#3A0E3E`), 칩(톤 요약 + `꿈 · 상징`), 축원문, 이미지 저장/공유하기/다른 꿈 해몽하기.
- **게이트 모달**: 神 마크 + "꿈 해몽은 손님을 기억해야 풀 수 있는 깊은 풀이라, 로그인 후에 홍연이 맞이할 수 있어요" — 기존 로그인 모달과 다른 전용 카피.

## 2. 데이터 계약 (프로토타입 → 서버 이관)

- **SYMBOLS 8종**: 뱀/물/이빨 빠짐/추락/시험/돈/불/하늘을 낢 — 각각 `meaning`(해석문) + `tone`(good/mixed/caution).
- **자동 감지 별칭**: 구렁이→뱀, "이가 빠"→이빨 빠짐, "날았"·"나는 꿈"→하늘을 낢. 감지 결과 최대 3개, 없으면 기본 '물'.
- **reading 로직**: 선택+감지 상징의 tone 집계 → good 우세/caution 우세/균형 3분기 → headline·overall·todayLink·chip(길몽·기회 / 액땜·정리 / 갈림길·선택).
- **세그먼트**: greeting → 상징별(1~3) → 전체 풀이 → 오늘 운세 연결 → 축원. 기존 운세와 동일한 단일 오디오 + 텍스트 길이 비례 경계 방식 재사용 가능.

## 3. 구현 매핑 (신 스택 기준 — 레거시 web는 동결)

| 프로토타입 | 구현 |
|---|---|
| 상징 감지·reading·세그먼트 | **backend/dream.py** (순수 함수 — 클라이언트 조작 방지 + 추후 LLM 교체 지점). POST `/api/dream/interpret` {text, symbols[]} → 로그인 게이트(401) |
| 오디오 | mock 톤 재사용 (`/audio/mock/…`, 세션 게이트 기존 적용). 실 TTS는 §3 hold와 동일 후속 |
| E0 카드 | S3TopicSelect에 dream 전용 카드 추가 (NEW 배지 + 금선 하이라이트) |
| 게이트 | 기존 LoginPrompt 재사용 + 전용 카피 prop |
| D1/D2/D3 | React 컴포넌트 신규 3종, 플레이어·Live2D moveTo·공유 시트·토스트 기존 모듈 재사용 |
| 프라이버시 | 꿈 원문: 서버 미저장(로그·이벤트에 원문 금지), 클라이언트도 플로우 이탈 시 state 초기화 |

## 4. 주의점

- 해몽 응답은 요청 바디 기반이라 fortuneId 캐시(부적 SVG)와 무관 — D3 "이미지 저장"은 서버 렌더러(share_card)가 꿈 카드를 지원할 때까지 정직한 목업(토스트)으로 둔다.
- `/api/dream/interpret`는 LLM 도입 시 비용이 가장 큰 엔드포인트가 될 예정 — rate limit(P0) 대상에 포함할 것.
- 이벤트: `dream_started`, `dream_interpreted`(상징 라벨만, 원문 금지), `dream_talisman_viewed` 추가.
