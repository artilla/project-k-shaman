## 3. 단위 경제 반영

기존 context-cache baseline:

```text
LLM 비용/건   ≈ $0.00155
TTS 비용/건   = 28s / 60 * $0.015 ≈ $0.00700
인프라/건      ≈ $0.00050
MISS 원가/건  ≈ $0.00905
```

TTS 실측 반영 후:

```text
LLM 비용/건   ≈ $0.00155
TTS 비용/건   = 31s / 60 * $0.015 ≈ $0.00775
인프라/건      ≈ $0.00050
MISS 원가/건  ≈ $0.00980
```

따라서 `unit-economics-simulator.xlsx`의 `TTS 신규합성 길이(초)/건` 기본값을 **28 → 31**로 갱신했다.

> **정정(2026-05-29)**: 위 "TTS 실측 반영 후" 블록의 `MISS 원가/건 ≈ $0.00980`은 **narration 유지(LLM $0.00155) 시점의 값**이다. 이후 v1.1 `scores_line` 채택으로 LLM 출력 토큰이 580→278로 줄어 LLM/건이 약 **$0.00079**가 되었고, **정본 MISS 원가는 `today-shindang-service-plan-v3.md` §13의 약 $0.0090**이다(시뮬레이터 재계산값 $0.00904). 본 §3의 $0.00980은 TTS 31초 보정 시점 기록으로 보존한다.

## 4. v1.1 판단 결과

TTS 실측만으로는 B 채택 여부를 결정하지 않는다. B의 비용 이득은 TTS가 아니라 LLM 출력 토큰 절감이며, 품질 리스크는 `scores` 문장 템플릿의 자연스러움에 있다.

청취 QA 결과는 `listening-decision-report.md`에 정리했다. 결론은 **`scores_line` 중간안 채택**이다.

- B scores 평균: 3.0
- B 연결 평균: 3.0
- B 전체 평균: 3.6

따라서 템플릿 기반 `scores` 문장은 채택하지 않고, LLM이 `scores_line` 한 문장만 출력한다. `narration` 배열은 v1.1에서 제거한다.
